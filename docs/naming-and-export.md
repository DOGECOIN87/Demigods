# Naming and Export Standard

## File naming

Use lowercase snake case:

`category_variantnumber_descriptive_tag.png`

Examples:

- `base_body_001_neutral_master.png`
- `hair_back_003_silver_long_wavy.png`
- `eyes_007_violet_starlight.png`
- `outfit_012_celestial_robe_white_gold.png`
- `hand_object_004_arcane_staff_pose_02_left.png`

Do not use character names.

## Export requirements

- Character-compatible assets: 1254 × 1254 transparent PNG, RGBA, sRGB.
- Backgrounds: 1254 × 1254 full-bleed opaque PNG or high-quality lossless WebP.
- Preserve exact full-canvas coordinates; never tightly crop traits.
- No checkerboard baked into transparency.
- No labels, guides, frames, watermarks, signatures, or neighboring assets.
- Remove all invisible guide bodies and unrelated pixels.
- Keep soft glow alpha without a solid rectangular backdrop.

## Quality control

Each asset must pass checks for dimensions, alpha, bounds, naming, category isolation, lighting direction, rig alignment, seam overlap, and compositing compatibility before approval.
