# Trait Candidate Intake

External co-created trait candidates belong here until QA and human approval pass.

- Create one subdirectory using the exact canonical category name.
- Upload one native 1254 × 1254 PNG per asset; batches of separate files are allowed.
- Use `_candidate_attempt_###.png` for failed or superseded attempts.
- Use `_candidate.png` only for the current passing review candidate.
- Never place an unapproved candidate under `assets/`.
- Never edit the production manifest for a candidate.
- Character-aligned prealignment candidates may use the locked coordinate guide before a base/pose is approved, but they still require genuine RGBA transparency and cannot be registered until the dependency composite passes.
- Recorded canvas-only translation, alpha cleanup, profile tagging, canonical renaming, and small non-redesign cleanup are allowed during intake; rescaling is not.
- Follow `prompts/19_individual_trait_asset_co_creation.md`.

Repository maintainers perform binary QA, rig overlays, composite testing, human approval, exact-hash registration, and production promotion after upload.
